## Summary

Flag `patient_index` ships **OFF**; enabled via DB reset (prod is test/seed only — no backfill).
Adds a non-identifying HMAC matching key (`patient_key`/`encounter_key`) over the three PHI modules,
a QPS-only key-only `patient_xref`, additive referral key transmission + a count-only receiver hint,
and a QPS cross-committee trajectory + access-audit view. **Pepper = locked-down `app.app_secrets`
table** (not Vault — empty at reset→seed; not GUC — needs superuser / cloud-blocked; hard-fail if
absent). Derivation triggers **always-on** (flag gates only RPCs/UI). Patient access audit emits
`patient.searched`/`patient.viewed` on the **global chain** via `app.audit_write` (key-only, never raw
MRN). Disposal: retain-marked-disposed; referrals **cascade-only** (no `dispose_referral_phi` —
follow-up). Single-tenant (no facility namespace).

## Follow-ups (non-blocking — see PROGRESS Follow-ups)

- `dispose_referral_phi` — referrals have no LGPD-erasure RPC (pre-existing Phase-22 gap); referral xref
  disposal is cascade-only today. The generic xref disposal mechanism plugs it in with no rework.
- **Pepper rotation** — `app.app_secrets.mrn_pepper` is a stable long-lived secret; rotating it orphans
  disposed-row keys (raw MRN gone). Documented residual; not built.
- **Case + referral QPS deep-link links** — need backend to export a public PQS-membership boolean
  (promote `isPqsMemberSelf()` to `src/lib/queries`), then FE gates the "ver trajetória" link on it.
- **Access-audit on the deep-link entry** — currently search-only (the audit query is MRN-keyed; the
  deep-link carries no identifier). Add an entity-keyed access-audit door for parity.
